## Schema
All shape tokens follow a variant naming scheme assigned to differing values.

### General Schema
| Variant | Description |
|---|---|
| `h1`, `h2`, `h3`, `h4` | Type variant for the different headers (h1-h4). |
| `body` | Type variant for all text. |
| `icon` | Type variant for icons. |
| `button` | Type variant for buttons. |
| `small` | Type variant for small texts (helper texts, etc.). |

### Special Schema (for the `family` token)
| Variant | Description |
|---|---|
| `global` | Type variant for all typography. |
| `header` | Type variant for the different headers (h1-h4). |
| `body` | Type variant for body text (paragraphs, spans, anchors, etc.). |
| `action` | Type variant for action texts (buttons, etc). |

## Token Presets
Here are the tokens pre-built in the design system.

| Token | Description |
|---|---|
| `family` | Token for setting font family values. |
| `size` | Token for setting font size values. |
| `weight` | Token for setting font weight values. |
| `line-height` | Token for setting line height values. |

## Using the tokens
```scss
hashi.theme-token-get('<type-token>-<type-variant>');
```
Example
```scss
@use 'path/to/@devprotocol/hashi';

.hs-component {
    @include hashi.theme-key-bind('component-label-size', hashi.theme-token-get('size-h1'));
}
```
Or in a component context:
```scss
@use 'path/to/@devprotocol/hashi/hs-component';

@include hs-component.render(
    $type: (
        family: 'global',
        size: 'h1',
        weight: 'h2'
    )
);
```
