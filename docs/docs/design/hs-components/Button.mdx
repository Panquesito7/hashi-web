# HSButton Component
## Picking a variant
When picking a button variant, you must always have the intent in mind. Hashi provides you 6 different states: text, outlined, filled, success, warning, and danger.

| Variant      | Description                                                                    | Preview                                                                                                           |
|--------------|--------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------|
| **Text**     | For actions that require the minimum amount of emphasis. Tertiary button.      | <button class="hs-button" role="button"><span class="hs-button__label">Button</span></button>                     |
| **Outlined** | For actions that require a good amount of emphasis. Secondary button.          | <button class="hs-button hs-button--outlined" role="button"><span class="hs-button__label">Button</span></button> |
| **Filled**   | For actions that require the maximum amount of emphasis. Primary button.       | <button class="hs-button hs-button--filled" role="button"><span class="hs-button__label">Button</span></button>   |
| **Success**  | For actions that indicate a positive change.                                   | <button class="hs-button is-success" role="button"><span class="hs-button__label">Button</span></button>          |
| **Warning**  | For actions that indicate a potential breaking change from the previous state. | <button class="hs-button is-warning" role="button"><span class="hs-button__label">Button</span></button>          |
| **Danger**   | For actions that indicate a breaking change from the previous state.           | <button class="hs-button is-danger" role="button"><span class="hs-button__label">Button</span></button>           |

## Color
- The button component must only have one primary color all throughout the design system.
- The ink colors must be contrasting from the fill color on filled variants.

## Shape
- Buttons must only use one radius, padding, margin, and spacing values throughout the design system.

## Typography
- The button label and ink size, weight, line height, and family must be consistent throughout the design system.
  - You can use any icon library you wish to use. However, you can only use one for consistency.
